# Contributing to Aglyn Docs

This is a standalone [Docusaurus](https://docusaurus.io) app. It has its **own**
`node_modules` (React 18) so it never conflicts with the monorepo root (React 19).

## Local development

```bash
cd apps/docs
npm install          # first time only
npm run start        # dev server with hot reload at http://localhost:3000
npm run build        # production build — fails on any broken internal link
npm run serve        # serve the production build locally
```

From the monorepo root you can also use the Nx wrapper:

```bash
npx nx serve docs
npx nx build docs
```

## Adding a page (the important part)

The sidebar is **autogenerated** from the folder structure — you never edit
`sidebars.ts`. To add a page:

1. Drop a Markdown/MDX file into the right folder under `docs/`.
2. Give it frontmatter:

   ```md
   ---
   sidebar_position: 2
   title: My page title
   description: One sentence for SEO and search.
   ---
   ```

3. Category label and order come from the folder's `_category_.json`.

Start from the canonical template at [`templates/feature-page.md`](templates/feature-page.md).
It encodes our house structure: **Overview → Plan availability → What you can do →
How to use it → Tips → Related**.

> Files whose name starts with `_` (e.g. `_category_.json`) are treated as partials
> and are **not** turned into pages.

## Plan-gating convention

Aglyn features are gated by plan. Flag it with an admonition right under the H1:

```md
:::info Plan availability
**Pro**. 1 dataset on Free; unlimited on Pro and above.
:::
```

Use `Free`, `Pro`, or `Business` to match the entitlement tiers.

## Theme & colors

The Material theme is a faithful port of the **console** app's MUI theme. The source
of truth is [`libs/shared/ui/theme/src/lib/console.theme.ts`](../../libs/shared/ui/theme/src/lib/console.theme.ts).
All colors live as CSS variables in [`src/css/custom.css`](src/css/custom.css). If the
console palette changes, update these to match:

| Docusaurus usage | CSS variable | console.theme.ts | Value |
| --- | --- | --- | --- |
| Accent / links / primary buttons | `--ifm-color-primary` | `secondary.main` | `#00b0ff` |
| Navbar + footer (light) | `--ifm-navbar-background-color` | `primary.main` (light) | `#404C5C` |
| Navbar (dark) | `--ifm-navbar-background-color` | `primary.main` (dark) | `#2C3540` |
| Footer (dark) | `--ifm-footer-background-color` | `surface.main` (dark) | `#202934` |
| Page background (light) | `--ifm-background-color` | `background.default` | `#F5F5F5` |
| Surface (light) | `--ifm-background-surface-color` | `background.paper` | `#FFFFFF` |
| Page background (dark) | `--ifm-background-color` | `background.default` (dark) | `#161c21` |
| Surface (dark) | `--ifm-background-surface-color` | `background.paper` (dark) | `#2a3440` |
| Info / Success / Warning / Error | `--ifm-color-*` | `info`/`success`/`warning`/`error` | `#1e88e5` / `#4CAF50` / `#FFAB40` / `#E53935` |
| Corner radius | `--ifm-global-radius` | `shape.borderRadius` | `4px` |

Typography is self-hosted **Roboto** / **Roboto Mono** via `@fontsource` (no external
CDN), matching Material.

## Images

Put screenshots in `static/img/<area>/` and reference them as `/img/<area>/file.png`.

## Deploying

The site deploys to Vercel using [`vercel.json`](vercel.json) (framework
`docusaurus-2`, output `build/`). Linking the Vercel project is a one-time setup step
done outside this repo.
